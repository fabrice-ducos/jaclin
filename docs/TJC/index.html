TJC Compiler Manual

Usage:
tjc module.tjc ?module.tjc ...?

What is TJC:

TJC is a Tcl to Java converter and compiler. TJC accepts Tcl source code and translates to Java source code. TJC scans Tcl files looking for proc declarations and converts each proc declaration to a Java class that implements the equivalent operations. The generated Java source code is then compiled to Java class files and saved in a JAR file. This JAR file can easily be loaded into Jacl (Tcl interpreter written in Java).

How it works:

The tjc compiler loads JDK configuration information from a file named jdk.cfg. This JDK configuration file must be saved in the bin/ directory where tjc executable is installed. See JDK Config File for more info about the contents of a JDK config file.

The user invokes the tjc executable and passes a .tjc module file on the command line. Processing a module file generates a pair of jar files. The first JAR file contains compiled Java class files and modified Tcl scripts. The second JAR file contains generated Java source code and unmodified Tcl scripts. For example, a module file xyz.tjc will generate xyzsrc.jar and xyz.jar. A TJC module file indicates which Tcl source files will be read and compiled by tjc. See Module File for more info about the contents of a module file.

A set of Tcl scripts and compiled proc implementations packaged inside a JAR file is known as a tjc package. The TJC::package Tcl command is provided to load a TJC package into the Jacl shell. See TJC package manual for more info.
